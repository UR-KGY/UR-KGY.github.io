## Unity 6 에서 처음 본 경고문
findobjectoftype 메서드를 사용 중 경고문 발생
findobjectoftype<> 은 씬 전체를 뒤지고 내부 정렬 과정까지 거쳐 속도가 느림
따라서 폐지될 예정으로 하고 최신버전에서는 FindFirstObjectByType<>()  , FindAnyObjectByType<>() 으로 사용을 권장
FindFirstObjectByType<>() : 씬 전체에서 제네릭 안 타입의 오브젝트를 찾을 때 순서상 가장 첫번째로 발견되는 오브젝트를 반환
FindAnyObjectByType<>() : 씬에서의 정렬 순서와 관계없이 제네릭 안 타입의 오브젝트 중 가장 먼저 탐색되는 것을 반환 (싱글톤 사용을 목적으로 한다면 성능이 가장 좋음)

## Steamworks 학습중

SteamManager -> 스팀 클라이언트를 초기화하고 관련 기능을 제공하는 싱글톤 . 다른 스팀 기능들을 사용하기 위해선 스팀 클라이언트 초기화가 필요한데 여기서 초기화하고 관련 기능들을 사용할 때 이 클래스를 경유하도록 수정 예정
핵심 메서드 
SteamClient.Init(); : 스팀 클라이언트 초기화 메서드 내컴퓨터의 스팀과 연동
SteamClient.RunCallbacks(); : 스팀에서 처리한 요청을 받아옴 주로 업데이트 문에서 지속적인 호출
SteamClient.Shutdown(); : 스팀 클라이언트와 스팀의 연동을 끊음  OnApplicationQuit 에서 처리중

## 리팩토링 진행 중 

이전의 SteamFriendManager 와 SteamLobbyManager 의 경우  실질적으로 프로필과 친구초대창의 UI 관련 기능들로 구성됨   LobbyManager 의 경우는 각 로비 이벤트 연결 및 로비 만들기 , 초대 등의 기능이 포함되어 있지만 SteamFriendManager 의 경우는 명칭과 기능이 부합하지 않음
-> UIManager 를 만들고 해당 클래스 명칭을 변경하거나 기능 이관 후 다른 용도의 클래스로 사용 예정


## Layout group 컴포넌트 학습중
layout 그룹 컴포넌트의 control child size는 해상도에 맞춰서 자식의 크기를 조절하는 역할을 수행함 child force expand 는 자식 오브젝트의 갯수에 따라 n등분하여 공간을 분배
layout element 의 flexible 옵션은 부모의 layout 그룹 컴포넌트가 활성화 되어 있을 때 어느 지분 만큼 자리를 먹겠다는 의미
예) 0 은 남는 자리 먹겠다 1은 있는대로 다먹겠다 0.2 0.4 이런식으로 쓰면 2/10 4/10 만큼 먹는 듯함
(캔버스는 scale with screen size 옵션이라 다른 옵션이면 해당하지 않을지도)
