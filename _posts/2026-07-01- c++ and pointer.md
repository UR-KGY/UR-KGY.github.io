# 📌 C# 개발자 관점의 C++/언리얼 포인터 구조 정리

## 1. 포인터와 역참조의 핵심 메커니즘
* **포인터 변수 (`Type* ptr`)**
  * 메모리 주소값 자체를 저장하는 바구니임.
  * C#의 클래스(참조 타입) 변수가 내부적으로 주소를 가리키는 원리와 완벽히 일치함.
* **주소 연산자 (`&`)**
  * 일반 변수의 실제 메모리 주소를 추출할 때 사용함.
* **역참조 연산자 (`*` 또는 `->`)**
  * 포인터가 가리키는 주소로 직접 찾아가 알맹이 데이터(값)를 꺼내거나 수정하라는 실시간 명령임.
  * `(*m).hp` 표기의 번거로움을 줄이기 위해 축약한 문법이 화살표 연산자(`m->hp`)이며, 내부에 이미 역참조 기능이 포함되어 있음.

---

## 2. 일반 메서드 반환 vs 포인터/참조자 활용 비교

C++은 포인터 없이 데이터를 반환하면 기본적으로 '통째로 복사'되는 비효율이 발생하므로 데이터의 종류에 따라 처리 방식을 다르게 가져감.

| 데이터 종류 | C# (유니티) 처리 방식 | C++ (언리얼) 처리 방식 | 메커니즘 특징 |
| :--- | :--- | :--- | :--- |
| **클래스 / 오브젝트**<br>(Actor, Component) | 일반 `return` 사용<br>(자동 참조 반환) | **포인터 `return` 사용**<br>(함수 타입 뒤에 `*` 명시) | 객체의 주소값(8바이트)만 가볍게 배달하여 원본을 제어함. |
| **구조체 / 대용량 데이터**<br>(FVector, FStruct 등) | 일반 `return` 사용<br>(값 복사 비용이 적음) | **참조자(`&`) 매개변수 사용**<br>(함수 인자로 빈 바구니 전달) | 복사 비용을 아끼기 위해 `return` 대신 매개변수로 주소를 받아 함수 내부에서 원본을 직접 채움. |
| **기본 타입**<br>(int, float, bool 등) | 일반 `return` 사용 | **일반 `return` 사용** | 데이터 크기가 매우 작으므로 포인터 없이 직설적으로 값을 복사해 반환함. |

## 3. 언리얼 엔진 클래스 간 참조 구조 (예시: Character - Weapon)

언리얼 엔진의 오브젝트는 C#처럼 항상 힙(Heap) 메모리에 생성되며, 클래스 간 데이터 교환 시 포인터를 매개체로 사용함.



* **관계 형성 (변수 선언)**
  * `MyCharacter` 클래스 내부에 무기 클래스의 주소를 담을 포인터 변수(`AWeapon* CurrentWeapon`)를 멤버로 선언하여 두 클래스를 연결함.
* **포인터 반환 함수 (`GetWeapon()`)**
  * 해당 함수는 별도의 매개변수를 받지 않음.
  * 캐릭터가 내부적으로 들고 있던 무기의 주소값(`CurrentWeapon`)을 그대로 외부에 배달해 주는 다리 역할을 수행함.
* **외부에서의 활용 및 값 변경**
  * 애니메이션 등 외부 시스템에서 `Character->GetWeapon()`을 통해 무기의 주소를 안전하게 받아옴.
  * 받아온 포인터 변수 뒤에 화살표 연산자를 붙여 `WeaponPtr->Damage = 50;` 형태로 무기 내부의 실제 데이터를 직접 변경함.

## 📌 최종 요약
> C++과 언리얼 엔진에서의 포인터는 C#의 클래스 참조 개념과 본질이 같음. 다만 문법적으로 포인터 변수 선언 시에는 **`*`**, 주소로 찾아가 값을 바꿀 때는 **`->`**를 명시하여 메모리를 제어하는 것임.
