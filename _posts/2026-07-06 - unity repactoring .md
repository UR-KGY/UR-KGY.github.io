## 이벤트 아키텍처 및 초기화 (Event System)

static이 없는 일반 UnityEvent 필드는 유니티 인스펙터 창에 노출되며, 유니티의 직렬화(Serialization) 시스템이 에디터 실행 시 자동으로 메모리에 인스턴스를 올려주기에 오류가 발생하지 않았음 하지만 실행시점을 고려해서 static 으로 선언 시엔 메모리에 직접적으로 올려놓기에 초기화 되지않음 따라서 선언 시 new 인스턴스 생성을 통해 초기화


## 리스너(Listener) / 핸들러(Handler) 중복 및 누수 방지
이벤트의 등록과 해제 시점이 어긋나면 파괴된 오브젝트를 참조하는 메모리 누수(유령 참조) 및 다중 등록으로 인한 스크립트 중복 실행 버그가 터짐 따라서 OnEnable(), OnDisable() 로 활성 비활성 때에 이벤트 구독을 관리

## 기술 용어 사전 

* 대리자 (Delegate): 메서드를 안전하게 담을 수 있는 주파수 규격이자 타입 그릇을 의미 (예: Action 키워드 등)
* 이벤트 (Event / UnityEvent): 대리자를 기반으로 신호를 외부에 송출하는 발행 주체입니다. 외부에서 마음대로 호출(Invoke)할 수 없고 오직 구독과 해제만 허용하도록 안전장치가 추가된 클래스 멤버 변수. UnityEvent는 여기에 유니티 인스펙터 직렬화 기능이 추가된 형태
* 리스너 / 핸들러 / 콜백 (Callback): 이벤트 신호가 발생했을 때 호출되는 타겟 메서드(함수)들을 통칭합니다. 

### 리스너(Listener) vs 핸들러(Handler)의 뉘앙스 차이
실무에서는 사실상 동의어로 혼용되나 명확한 관점 차이가 존재
* 리스너: 신호가 오기를 "귀 기울여 기다리는 상태나 관계"에 초점을 맞춘 표현. 유니티 API 스타일(AddListener)에서 주로 찾아볼 수 있음
* 핸들러: 신호가 도착했을 때 이를 "맡아서 처리하는 구체적인 행동이나 로직"에 초점을 맞춘 표현. C# 스타일에서 관례적으로 메서드명 뒤에 Handler 문구를 명명할 때 자주 쓰임.
